# User Stories

## Назначение документа

Документ содержит пользовательские истории первой версии Smart Home Management Platform.

User Stories описывают функциональность системы с точки зрения конечного пользователя и используются при планировании разработки.

---

## Правила оформления

Каждая пользовательская история:

- имеет уникальный идентификатор;
- описывает ценность для пользователя;
- связана с функциональными требованиями;
- содержит критерии приемки.

---

# US-001 Регистрация пользователя

**Приоритет:** Высокий

**Связанные требования:** FR-001, FR-002, FR-003

### User Story

Как новый пользователь

Я хочу зарегистрироваться в системе

Чтобы получить возможность управлять своим умным домом.

### Критерии приемки

- пользователь может создать учетную запись;
- обязательные поля проходят проверку;
- после успешной регистрации пользователь может войти в систему.

---

# US-002 Создание дома

**Приоритет:** Высокий

**Связанные требования:** FR-005

### User Story

Как пользователь

Я хочу создать свой дом

Чтобы разделить устройства по объектам недвижимости.

### Критерии приемки

- пользователь может создать дом;
- пользователь может указать название дома;
- новый дом появляется в списке домов.

---

# US-003 Создание комнаты

**Приоритет:** Высокий

**Связанные требования:** FR-008

### User Story

Как пользователь

Я хочу создавать комнаты

Чтобы удобно группировать устройства.

### Критерии приемки

- пользователь может добавить комнату;
- можно изменить название комнаты;
- комната отображается в выбранном доме.

---

# US-004 Подключение устройства

**Приоритет:** Высокий

**Связанные требования:** FR-011

### User Story

Как пользователь

Я хочу подключить новое устройство по QR-коду

Чтобы быстро начать управление устройством.

### Критерии приемки

- приложение открывает камеру;
- QR-код успешно считывается;
- устройство появляется в списке.

---

# US-005 Управление устройством

**Приоритет:** Высокий

**Связанные требования:** FR-014, FR-015

### User Story

Как пользователь

Я хочу включать и выключать устройство

Чтобы управлять состоянием оборудования из мобильного приложения.

### Критерии приемки

- пользователь может изменить состояние устройства;
- новое состояние отображается в интерфейсе;
- изменение сохраняется после обновления страницы.
