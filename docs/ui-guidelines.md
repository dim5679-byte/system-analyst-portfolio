# UI Guidelines

This document defines the visual and structural rules for the System Analyst Portfolio.

## Purpose

The portfolio should look like a professional product documentation, not just a file storage.

## General Style

- Minimalistic
- Structured
- Easy to scan
- Consistent across all pages
- Employer-oriented

## Page Structure

Each main page should follow this logic:

1. Title
2. Short description
3. Navigation links
4. Main content
5. Related materials

## Typography

- Use one `# H1` heading per page.
- Use `## H2` for main sections.
- Use `### H3` for subsections.
- Avoid long paragraphs.
- Prefer short blocks of text.

## Tables

Use tables for structured information:

| Item | Description | Link |
|---|---|---|
| Example | Short explanation | View |

## Project Page Template

Each project should include:

1. Overview
2. Business Context
3. Goal
4. My Role
5. Tasks
6. Solution
7. Artifacts
8. Result
9. Lessons Learned

## Artifact Page Template

Each artifact page should include:

1. Purpose
2. Context
3. Artifact Preview
4. Explanation
5. Related Project
6. Link to Source File

## Images and Diagrams

Each diagram should have:

- clear title;
- short explanation;
- readable resolution;
- consistent naming.
